## 一、领域事件消费 — 操作日志闭环（Feature B）

### 改动说明

打通了 Tycho 之前埋好的 6 个领域事件（CustomerCreatedEvent、ContactAddedEvent、ContractCreatedEvent、ContractCompletedEvent、ContractCancelledEvent、PaymentRecordedEvent），实现保存聚合后自动将事件翻译为操作日志并持久化。

### 新增文件

| 文件 | 作用 |
|------|------|
| `CRM.Domain/Events/IDomainEventDispatcher.cs` | 事件分发器接口 |
| `CRM.Domain/OperationLogs/OperationLog.cs` | 操作日志实体（继承 AggregateRoot\<int\>） |
| `CRM.Infrastructure/Events/DomainEventDispatcher.cs` | 6 种事件 → OperationLog 的 switch 映射实现 |
| `CRM.Application.Contracts/OperationLogs/Dtos/OperationLogDto.cs` | 输出 DTO |
| `CRM.Application.Contracts/OperationLogs/Dtos/OperationLogQueryDto.cs` | 查询 DTO（支持实体名/日期/分页筛选） |
| `CRM.Application.Contracts/OperationLogs/IOperationLogAppService.cs` | 服务接口 |
| `CRM.Application/OperationLogs/OperationLogAppService.cs` | 分页查询实现 |
| `CRM.Web/Controllers/Api/OperationLogsApiController.cs` | `GET /api/operation-logs` |
| `CRM.Client/src/types/operationLog.ts` | 前端类型 |
| `CRM.Client/src/api/operationLogApi.ts` | 前端 API 封装 |
| `CRM.Client/src/pages/operationLogs/OperationLogPage.tsx` | 时间线风格展示页面 |

### 修改文件

| 文件 | 改动 |
|------|------|
| `CRM.Infrastructure/Repositories/EfRepository.cs` | 注入 IDomainEventDispatcher，Insert/Update 后调用 DispatchEventsAsync |
| `CRM.Infrastructure/Persistence/CrmDbContext.cs` | 新增 OperationLogs DbSet + Fluent API 配置 |
| `CRM.Web/Program.cs` | 注册 IDomainEventDispatcher + IOperationLogAppService |
| `CRM.Client/src/App.tsx` | 新增 `/operation-logs` 路由 |
| `CRM.Client/src/components/AppNavbar.tsx` | 导航栏新增"操作日志"入口 |
| `CRM.Client/src/index.css` | 新增时间线样式 |

### 核心流程

```text
聚合根.AddDomainEvent()
  → EfRepository.InsertAsync/UpdateAsync 保存主数据
  → DispatchEventsAsync() 收集事件
  → DomainEventDispatcher 映射为 OperationLog
  → 持久化到 OperationLogs 表
```

## 二、客户等级智能评估（Feature A）

### 改动说明

基于三维度评分自动评定客户等级：合同金额（40 分）、回款及时性（40 分）、满意度（20 分）。总分 100 分，映射到 4 个等级：战略客户（≥80）、重点客户（≥60）、普通客户（≥40）、风险客户（<40）。

### 新增文件

| 文件 | 作用 |
|------|------|
| `CRM.Domain.Shared/Enums/CustomerLevel.cs` | 等级枚举：Strategic / Important / Normal / Risk |
| `CRM.Domain/Customers/CustomerEvaluationResult.cs` | 评估结果值对象（三维度得分 + 等级 + 建议文字） |
| `CRM.Domain/Customers/CustomerEvaluationService.cs` | 领域服务：跨聚合计算评分 |
| `CRM.Application.Contracts/Customers/Dtos/CustomerEvaluationDto.cs` | 评估结果 DTO |
| `CRM.Client/src/pages/customers/CustomerEvaluationModal.tsx` | 评估弹窗组件（进度条展示三维度得分） |

### 修改文件

| 文件 | 改动 |
|------|------|
| `CRM.Domain/Customers/Customer.cs` | 新增 `Level` 属性（默认 Normal）+ `SetLevel()` 方法 |
| `CRM.Application.Contracts/Customers/Dtos/CustomerDto.cs` | 新增 Level 字段 |
| `CRM.Application.Contracts/Customers/ICustomerAppService.cs` | 新增 `EvaluateAsync(int customerId)` |
| `CRM.Application/Customers/CustomerAppService.cs` | 实现 EvaluateAsync（调域服务 + 更新等级 + MapToDto 补 Level） |
| `CRM.Web/Controllers/Api/CustomersApiController.cs` | 新增 `GET /api/customers/{id}/evaluation` |
| `CRM.Client/src/types/customer.ts` | 新增 level 字段 + CustomerEvaluationDto 接口 |
| `CRM.Client/src/api/customerApi.ts` | 新增 `evaluate(id)` 方法 |
| `CRM.Client/src/pages/customers/CustomerListPage.tsx` | 等级筛选（4 级下拉）+ 评估弹窗集成 |
| `CRM.Client/src/components/customers/CustomerTable.tsx` | 等级 Badge（四色）+ "评估"按钮 |

### 评分规则

```text
合同金额维度（40 分）：
  ≥100万→40 | ≥50万→32 | ≥10万→24 | ≥5万→16 | ≥1万→8 | <1万→4

回款及时性（40 分）：
  按时率≥90%→40 | ≥70%→32 | ≥50%→24 | ≥30%→16 | <30%→8

满意度（20 分）：
  当前固定 15 分（后续可扩展为客户反馈数据）
```

## 三、现金流预测看板（Feature E）

### 改动说明

纯查询服务，按月聚合 PaymentPlan 的计划回款金额与实际回款金额，前端以纯 CSS 分组柱状图 + 汇总卡片 + 月度明细表展示。无 Domain 层变更。

### 新增文件

| 文件 | 作用 |
|------|------|
| `CRM.Application.Contracts/Contracts/Dtos/CashFlowForecastDto.cs` | 月度预测 DTO（含 Difference / AchievementRate 计算属性） |
| `CRM.Application.Contracts/Contracts/ICashFlowForecastService.cs` | 预测服务接口 |
| `CRM.Infrastructure/Services/CashFlowForecastService.cs` | GROUP BY 月份聚合 + 补全空白月份 |
| `CRM.Client/src/types/cashFlow.ts` | 前端类型 |
| `CRM.Client/src/api/cashFlowApi.ts` | 前端 API 封装 |
| `CRM.Client/src/components/dashboard/CashFlowChart.tsx` | 纯 CSS 分组柱状图 + 汇总 + 明细表 |

### 修改文件

| 文件 | 改动 |
|------|------|
| `CRM.Infrastructure/CRM.Infrastructure.csproj` | 新增对 Application.Contracts 的项目引用 |
| `CRM.Web/Controllers/Api/DashboardApiController.cs` | 注入 ICashFlowForecastService，新增 `GET /api/dashboard/cashflow-forecast` |
| `CRM.Web/Program.cs` | 注册 ICashFlowForecastService → CashFlowForecastService |
| `CRM.Client/src/pages/dashboard/DashboardPage.tsx` | 嵌入 CashFlowChart 组件 |
| `CRM.Client/src/index.css` | 新增 .cashflow-chart / .cashflow-bars / .cashflow-bar 样式 |

### 查询逻辑

- 查询未来 12 个月 PaymentPlans
- `GROUP BY Year, Month` → `SUM(PlanAmount)`, `SUM(ActualAmount)`
- 补全没有数据的月份（确保图表连续）
- 前端达成率颜色编码：≥80% 绿 / ≥50% 黄 / <50% 红
