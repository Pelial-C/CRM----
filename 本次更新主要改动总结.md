# 本次更新主要改动总结

更新分支：`Tycho_Code_Domain-modeler`

最新功能提交：`464951c 完善合同模块页面与领域规则闭环`

## 一、合同模块 Web 页面补齐

新增合同管理页面目录：`CRM.Lite/CRM.Web/Views/Contract`

主要页面：

- `Index.cshtml`：合同列表、关键字搜索、状态筛选、详情/编辑/作废操作。
- `Create.cshtml`：新增合同、客户下拉、联系人联动、动态合同明细、小计自动计算。
- `Edit.cshtml`：编辑合同、回显客户/联系人/状态/明细，并保留联系人联动。
- `Detail.cshtml`：合同详情、合同明细、回款计划列表、自动生成计划、新增计划、登记回款。

新增控制器：

- `CRM.Lite/CRM.Web/Controllers/ContractController.cs`

支持：

- 合同列表查询。
- 合同新增、编辑、详情。
- 合同作废。
- 客户联系人联动接口：`/Contract/GetContactsByCustomerId`
- 自动生成回款计划。
- 新增回款计划。
- 登记实际回款。

## 二、客户下拉与联系人联动

新增 DTO：

- `CRM.Lite/CRM.Application.Contracts/Customers/Dtos/CustomerSelectDto.cs`
- `CRM.Lite/CRM.Application.Contracts/Customers/Dtos/CustomerQueryDto.cs`

## 三、合同 DTO 后端校验

修改 DTO：

- `CreateContractDto`
- `UpdateContractDto`
- `ContractItemDto`

新增 DTO：

- `AddPaymentPlanDto`
- `RecordPaymentDto`
- `ContractListQueryDto`
- `PagedResultDto<T>`

增加校验：

- 合同编号必填、最大 50。
- 合同名称必填、最大 100。
- 客户必须选择。
- 合同金额必须大于 0。
- 结束日期不能早于开始日期。
- 预警天数范围 0 到 365。
- 明细产品名称必填。
- 数量必须大于 0。
- 单价必须大于等于 0。
- 回款计划金额和实际回款金额必须大于 0。

## 四、合同领域规则闭环

修改：

- `CRM.Lite/CRM.Domain/Contracts/Contract.cs`
- `CRM.Lite/CRM.Domain/Contracts/ContractStatus.cs`
- `CRM.Lite/CRM.Domain/Contracts/PaymentPlan.cs`

补齐规则：

- 新建合同默认为草稿。
- 草稿可进入执行中。
- 执行中可完成。
- 草稿和执行中可作废。
- 执行中可终止。
- 已作废、已终止、已完成合同不允许随意编辑。
- 已作废、已终止合同不允许登记回款。
- 所有回款计划结清后合同自动完成。
- 回款计划逾期状态可刷新。

## 五、合同查询性能优化

修改：

- `CRM.Lite/CRM.Domain/Repositories/IRepository.cs`
- `CRM.Lite/CRM.Infrastructure/Repositories/EfRepository.cs`
- `CRM.Lite/CRM.Application/Contracts/ContractAppService.cs`

完成：

- Repository 增加 `IQueryable<TEntity> Query()`。
