## 一、 DDD 设计原则与架构约定

本项目采用领域驱动设计（DDD）思想，结合 8 天集中实训的实际情况，制定以下架构约定：
1.  **架构分层**：采用经典四层架构（Domain 领域层、Application 应用层、Infrastructure 基础设施层、Web 展现层）。
2.  **适度充血模型**：摒弃传统的贫血模型。实体和聚合根不仅包含数据属性，还必须封装**核心状态变更、基础业务规则校验和内部集合管理**。应用服务（AppService）仅负责流程编排和跨聚合协调。
3.  **聚合边界严格**：跨聚合引用只能通过 ID（或 ID+名称冗余），严禁在聚合根内部直接持有另一个聚合根的实体对象引用。

## 二、 战略设计：限界上下文划分 (Bounded Contexts)

根据业务职责，系统划分为以下两个核心限界上下文：

### 2.1 客户上下文 (Customer Context)
*   **核心职责**：管理企业客户的基本信息、联系人网络以及客户地址等档案数据。
*   **关注点**：客户资产的建立与维护，保证客户数据的完整性和唯一性。

### 2.2 合同上下文 (Contract Context)
*   **核心职责**：管理合同的签订、明细记录以及后续的回款计划与执行状态。
*   **关注点**：商业契约的履约过程、资金回款进度的追踪。

*(注：基础数据如“行业分类”等，初期作为枚举或简单字典处理，不单独划分为上下文，以降低初期开发复杂度。)*

## 三、 战术设计：聚合、实体与值对象

### 3.1 客户聚合 (Customer Aggregate)

客户聚合负责管理客户及其直接关联的联系人和地址信息。

*   **聚合根 (Aggregate Root)**：`Customer` (客户)
    *   **属性**：Id, Name (企业名称), CreditCode (统一社会信用代码), Industry (行业), Remark (备注), CreationTime。
    *   **充血行为**：`AddContact()`, `RemoveContact()`, `UpdateAddress()`。
*   **实体 (Entity)**：`Contact` (联系人)
    *   *说明*：联系人拥有独立的 Id，但其生命周期依附于客户，不能脱离客户独立存在。
    *   **属性**：Id, Name, Title (职务), Phone, Email, IsKeyDecisionMaker (是否关键决策人)。
*   **值对象 (Value Object)**：`Address` (地址)
    *   *说明*：地址没有独立的生命周期标识（无 Id），仅作为客户的一个属性存在。在 EF Core 中通过 `Owned Entity` 映射。
    *   **属性**：Province (省), City (市), District (区), Detail (详细地址)。

### 3.2 合同聚合 (Contract Aggregate)

合同聚合负责管理合同本体、产品/服务明细以及回款计划。

*   **聚合根 (Aggregate Root)**：`Contract` (合同)
    *   **属性**：Id, ContractNo (合同编号), Name (合同名称), SignDate (签订日期), TotalAmount (总金额), Status (状态), CustomerId (关联客户ID), CustomerName (关联客户名称-冗余), ContactId (关联联系人ID)。
    *   **充血行为**：`AddItem()`, `AddPaymentPlan()`, `RecordPayment()`, `Terminate()`。
*   **实体 (Entity)**：`ContractItem` (合同明细)
    *   **属性**：Id, ProductName (产品/服务名称), Quantity (数量), UnitPrice (单价), Subtotal (小计)。
*   **实体 (Entity)**：`PaymentPlan` (回款计划)
    *   **属性**：Id, PlanDate (计划回款日期), PlanAmount (计划金额), ActualAmount (实际回款金额), ActualDate (实际回款日期), Status (状态：未回款/部分/已结清), Description (期数说明)。

## 四、 上下文映射 (Context Mapping)

**合同上下文** 与 **客户上下文** 之间存在“客户-供应商 (Customer-Supplier)”或“发布语言 (Published Language)”的映射关系。

*   **设计决策**：在 `Contract` 聚合根中，**不直接引用 `Customer` 和 `Contact` 对象**。
*   **实现方式**：`Contract` 仅保存 `CustomerId`、`CustomerName` 和 `ContactId`。
*   **优势**：实现了上下文之间的物理和逻辑解耦。合同创建时，通过应用层（AppService）查询客户上下文获取名称并冗余保存，避免了 EF Core 中复杂的跨 DbContext 联表查询和级联更新问题。

## 五、 充血模型行为设计指南 (致后端开发)

为了确保 Day 3 和 Day 4 的代码符合 DDD 规范，请后端开发在编写实体类时，严格遵循以下设计模式：

### 5.1 保护内部状态 (Encapsulation)
*   所有实体的属性 `set` 访问器应尽可能设为 `private` 或 `protected`，防止外部直接篡改数据。
*   集合属性（如 `Contacts`, `Items`）应使用私有后备字段（Backing Field），对外仅暴露 `IReadOnlyCollection<T>`。

### 5.2 聚合根负责一致性校验
*   业务规则（如：回款总额不能大于合同总额、联系人数量限制）必须在聚合根的方法内部进行校验，抛出领域异常（Domain/Business Exception），而不是在 Controller 或 AppService 中校验。

### 5.3 值对象的不可变性 (Immutability)
*   值对象（如 Address）一旦创建，其属性不应被修改。如果需要修改地址，应在聚合根中替换整个值对象实例。

### 2. 合同聚合根设计 (充血模型)
*   public class Contract : AggregateRoot<int> {
        // --- 基础属性 (私有 set 保护状态) ---
        public string ContractNo { get; private set; }
        public string ContractName { get; private set; }
        public string CabinetNo { get; private set; } // 合同柜编号
        
        // 关联客户 (甲方) - 跨聚合引用只存ID和名称
        public int CustomerId { get; private set; }
        public string PartyAName { get; private set; } // 合同甲方名称
        
        // 关联联系人
        public int ContactId { get; private set; }
        public string ContactName { get; private set; }

        // 组织归属
        public string RegionalCompany { get; private set; } // 区域公司
        public string AffiliatedCompany { get; private set; } // 所属公司

        // 业务属性
        public ServiceType ServiceType { get; private set; }
        public ContractType ContractType { get; private set; } // 新签/续签
        public DateTime StartDate { get; private set; }
        public DateTime EndDate { get; private set; }
        public int WarningDays { get; private set; } // 到期预警天数
        public decimal TotalAmount { get; private set; } // 合同期合同额
        public PaymentFrequency PaymentFrequency { get; private set; } // 约定到账频率

        // --- 内部集合 (使用私有后备字段，EF Core 支持映射) ---
        private readonly List<ContractItem> _items = new();
        public IReadOnlyCollection<ContractItem> Items => _items.AsReadOnly();

        private readonly List<PaymentPlan> _paymentPlans = new();
        public IReadOnlyCollection<PaymentPlan> PaymentPlans => _paymentPlans.AsReadOnly();

        // 保护无参构造函数 (EF Core 实例化需要)
        protected Contract() { }

        // --- 充血构造函数 (强制必填项) ---
        public Contract(
            string contractNo, string contractName, int customerId, string partyAName, 
            int contactId, string contactName, decimal totalAmount, 
            DateTime startDate, DateTime endDate, PaymentFrequency frequency) 
        {
            if (totalAmount <= 0) throw new BusinessException("合同金额必须大于0");
            if (endDate <= startDate) throw new BusinessException("终止时间必须晚于起始时间");

            ContractNo = contractNo;
            ContractName = contractName;
            CustomerId = customerId;
            PartyAName = partyAName;
            ContactId = contactId;
            ContactName = contactName;
            TotalAmount = totalAmount;
            StartDate = startDate;
            EndDate = endDate;
            PaymentFrequency = frequency;
            
            // 默认值初始化
            ContractType = ContractType.NewSign;
            WarningDays = 30; // 默认提前30天预警
            CabinetNo = "UNASSIGNED"; // 默认未分配柜子
        }

        // --- 充血行为方法 (业务逻辑封装在实体内) ---

        /// <summary>
        /// 添加合同明细
        /// </summary>
        public void AddItem(string productName, int quantity, decimal unitPrice) {
            var subtotal = quantity * unitPrice;
            _items.Add(new ContractItem(productName, quantity, unitPrice, subtotal));
        }

        /// <summary>
        /// 根据约定的到账频率，自动生成回款计划 (核心充血逻辑)
        /// </summary>
        public void GeneratePaymentPlans() {
            if (_paymentPlans.Any()) {
                throw new BusinessException("回款计划已生成，请勿重复操作。");
            }

            int months = (int)PaymentFrequency;
            DateTime currentDate = StartDate;
            decimal amountPerPeriod = TotalAmount / (12 / months); // 简单平摊计算

            while (currentDate < EndDate) {
                currentDate = currentDate.AddMonths(months);
                if (currentDate > EndDate) currentDate = EndDate; // 最后一期调整为终止时间

                _paymentPlans.Add(new PaymentPlan(currentDate, amountPerPeriod, $"第{_paymentPlans.Count + 1}期"));
            }
        }

        /// <summary>
        /// 记录实际回款 (更新状态)
        /// </summary>
        public void RecordPayment(int planId, decimal actualAmount, DateTime actualDate) {
            var plan = _paymentPlans.FirstOrDefault(p => p.Id == planId);
            if (plan == null) throw new BusinessException("未找到指定的回款计划");
            
            plan.RecordActualPayment(actualAmount, actualDate);
        }
    }
