# 领域模型说明

更新时间：2026-07-01

## 前后端架构说明

- 后端采用 ASP.NET Core Web API + EF Core + DDD 分层架构。
- 前端采用 React + Vite + TypeScript。
- `CRM.Web` 作为 API Host，提供 Swagger、CORS 和 REST API。
- `CRM.Client` 作为 React 前端，通过 Axios 调用 `/api/*` 接口。

## 聚合设计

### Customer 聚合

客户是客户管理模块的聚合根。联系人依附客户存在，不能脱离客户独立维护。

主要职责：

- 维护企业客户基础信息。
- 维护客户下联系人。
- 执行客户逻辑删除。
- 禁止已删除客户继续新增或维护联系人。

### Contract 聚合

合同是合同管理模块的聚合根。合同明细和回款计划依附合同存在。

主要职责：

- 维护合同基础信息和合同明细。
- 管理合同状态流转。
- 生成和维护回款计划。
- 登记实际回款并自动推进回款状态。
- 所有回款计划结清后自动完成合同。

## 领域规则清单

- 客户名不能为空且唯一。
- 客户有关联合同时不能物理删除，只能逻辑删除。
- 联系人不能脱离客户独立存在。
- 已删除客户不能新增联系人。
- 合同必须关联有效客户。
- 已删除客户不能用于新建合同。
- 合同编号必须唯一。
- 合同金额必须大于 0。
- 合同结束日期不能早于开始日期。
- 合同明细至少一项。
- 合同明细小计由后端根据数量和单价重新计算。
- 回款计划依附合同存在。
- 回款计划累计金额不能超过合同总金额。
- 实际回款累计不能超过合同总金额。
- 实际回款金额不能超过对应计划金额。
- 作废合同不能新增回款计划。
- 作废合同不能登记回款。
- 终止合同不能登记回款。
- 所有回款计划结清后合同自动完成。

## 合同状态流转说明

```text
Draft -> Executing -> Completed
Draft / Executing -> Cancelled
Executing -> Terminated
```

状态规则：

- 新建合同默认为 Draft。
- Draft 可以变为 Executing。
- Executing 可以变为 Completed。
- Draft 和 Executing 可以 Cancelled。
- Executing 可以 Terminated。
- Completed、Cancelled、Terminated 不允许随意回退。
- 状态变更必须通过领域方法完成，不能由前端直接覆盖。

## 回款计划状态流转说明

```text
Pending -> PartialPaid -> Paid
Pending / PartialPaid -> Overdue
```

状态规则：

- 登记部分金额后变为 PartialPaid。
- 登记金额达到计划金额后变为 Paid。
- 未结清且计划日期早于当前日期时变为 Overdue。
